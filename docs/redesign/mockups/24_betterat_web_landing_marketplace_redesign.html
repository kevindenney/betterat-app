BetterAt · Web redesign
Landing & Marketplace,
finally on the same rails

Today the public site has three discovery surfaces that never connect: landing interest cards route to /[interest], the marketplace lives at /marketplace, and they pull from different data sources. A visitor who clicks Drawing can't subscribe to “Learning to Draw.” This deck unifies them: the interest becomes the spine, the marketplace becomes the inventory.

Today — three silos

Separate routes, separate hooks, no path between them.

Landing interest card⤍ /[interest]

No interest card ever reaches the marketplace. “Learning to Draw” is unsellable from Drawing.

Redesign — one spine

Interest is the entry point; marketplace is the content, filtered.

Interest = /marketplace?interest=slug

Organizations — join✓
People — follow✓

One blueprint card component + one data source feed both the interest page and the global marketplace.

Filter = interest link

Clicking a filter chip is identical to landing on /[interest]. The interest page and the storefront are one component in two states.

One card component

The same <BlueprintCard> renders on landing, interest page, and storefront — author, interest pill, rating, price, Subscribe.

Cross-navigation

Each card's interest pill links back to the interest; each interest's “see all” links here. The loop closes.
