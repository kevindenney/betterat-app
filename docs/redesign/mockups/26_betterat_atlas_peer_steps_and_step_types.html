Atlas · Peer steps + the Step / Race model
Two changes, one design. (1) Peer steps stop hiding behind a “+N” badge — they de-cluster into tappable, relationship-colored pins with a drill-down sheet and a privacy-aware detail callout. (2) Steps lose the invented 5-kind taxonomy. A step is a step; the one flag that matters is “is this a race?”, because a race is what unlocks Atlas’s course, marks, and conditions.
ReframeWe don’t have step types — we have steps. Phase M derived race / practice / boat work / learn / coach from keywords the user never set. The map doesn’t need a 6-color legend. The only distinction that changes Atlas behavior is race vs. everything-else: a race carries a venue, a course, marks, and a wind/tide read; an ordinary step is just a located dot encoded by status (planned → done) and, for peers, by relationship (you / crew / fleet / following). So: one optional is_race flag, set by the sailor when planning, replaces the keyword resolver.
