Atlas, re-rigged for sail racers
The current Atlas treats the map as a generic pin board. Racers don't read a chart that way — they read it as a cockpit: where's the breeze, where's the current, which side pays, and what's my next move on the water. This redesign pulls the conditions + strategy out of the thin bottom strip into a legible cockpit card, declutters the chart so the course geometry (favored side, thirds, marks) reads at a glance, and fixes the broken Nearby surface (real names, deduped, tap-to-fly-to-map).

Per-surface assessment & what changed

What's broken today (from the screenshots)

  • FIXNearby shows raw emails, duplicated. Rows render demo-markus@regattaflow.app twice. Root cause: demo sailors' profiles.full_name holds the email, and the list isn't deduped per person. The atlas_peer_steps_near RPC already COALESCEs name fields — the fix is seeding real names into profiles + a per-sailor dedupe in DiscoverNearbyContent.
  • FIXTapping a nearby step opens the editor, not the map. DiscoverNearbyContent.tsx:164 hardcodes router.push('/step/{id}'). It should close the overlay and fly the chart to the step's lat/lng. Add an onNearbyStepPress(lat,lng) callback from AtlasScreen.
  • FIXStrategy + conditions buried in a 1-line strip. The single most valuable thing for a racer (which side pays + why) was a thin band. Promoted to the cockpit card.
  • FIXGeneric "Race 1 / 2 / 3" picker and "000° / 0° N" default wind. Racers can't tell courses apart; a 0° wind draws a nonsense course. Picker now names series + course type; wind defaults to the venue's live observation.
  • FIXMap clutter. Overlapping condition chips, uniform wind arrows with no pressure encoding, collided sheets. Decluttered to one top row + pressure-encoded arrows + a single bottom card.

What's already right (keep)

  • KEEPFavored-side green handedness — verified correct in the sim (tactical right reads screen-left on a north-up map under southerly wind; that's the sailor's upwind view, not a bug).
  • KEEPBOTTOM/MIDDLE/UPPER thirds + LEFT/RIGHT labels drawn on the chart, tied to the strategy card's per-segment advice.
  • KEEPAtlas as the universal "where" lens — the cockpit pattern generalizes (a nurse's clinical site, a runner's route) even though sailing is the first vertical.

The core move: chart → cockpit

  • NEWCockpit card replaces the strip: three domain-colored gauges (wind / current / sea) + a plain-language favored-side call + a per-third game plan. Drag to expand.
  • NEWPressure-encoded wind field — arrow length + opacity show where the breeze is, not a uniform grid.
  • NEWConsolidated top chrome (search + layer filter), edge-to-edge chart below — the queued Apple-Maps redesign.
  • NEWSlim next-step chip that never collides with the Nearby / picker sheets.
