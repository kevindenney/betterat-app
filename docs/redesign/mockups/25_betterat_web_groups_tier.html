Groups belong on the
interest page too

Mockup 24 gave the interest page four tiers — Blueprints, Organizations, People, About. But the unit people actually move through the work with is the group: a sailor's fleet, a nursing student's cohort. These are one primitive on three dials (lifespan · who-pays · authority), so they get one tier — Groups · join — sitting between Organizations and People. The web cards echo the app Library tab's canonical group rows: square mark, name, role, roster, join.

00Why one tier, not three

Lifespan

ephemeral ───────●─ standing

How long the group lives. A regatta crew dissolves in a weekend; a class fleet runs for years; a Spring cohort spans one term.

Renders as a chip: Standing · Seasonal · One-off

Who pays

members pay ──●──── payer owns seats

Members-pay (a fleet everyone chips into) vs. payer-owns-seats (a school buys cohort seats for students). Gates the enterprise-admin surface.

Drives whether a Join button or a roster invite shows

Authority

peer / opt-in ───●── assigned / led

Peer-run (sailors self-organize) vs. assigned (a coach leads, an instructor places you). Gates consent and the role descriptor.

Peer-run shows Join; coach-led shows Request to join

BetterAtGroups for interest = nursing · placed cohorts + open study cohorts, same tier as fleets

Same card, different dials

A fleet and a cohort are the identical card. Only the axis chips move: nursing leans program-owns-seats + faculty-led; sailing leans members-pay + peer-run.

Placed vs. open

Program-owned cohorts show Placed by program (no self-join). Open study cohorts — peer + members-subscribe — show a real Join. The CTA is computed from the axes.

Persona vocab on top

The tier label stays "Groups" but band copy speaks native: fleets/squads for sailors, cohorts/clinical groups for nurses. Same structure, the craft's words.

SQUARE MARK

Deterministic color from group id (pickSquareMarkColor) + initials — the same mark the app's Library renders, so a fleet looks identical across web and mobile.

KIND LINE

Persona-native noun (fleet / squad / cohort / clinical group) + class/program + place. Never the generic word "group".

AXIS CHIPS

The three dials made visible — lifespan, who-pays, authority. These are data, not layout. They also compute the footer CTA.

ROSTER

Overlapped avatars preview real members + count, echoing the Library "real cards, never bare rows" rule.

FOOTER

If you're in it → role descriptor (Crew / Student / Coach). If not → a CTA computed from the axes: Join (peer + members-pay), Request to join (coach-led), or Placed by program (payer owns seats).

Data reality

Fleets are first-class (fleetService); cohorts live under orgs (SampleCohort). useUserFleets is global, not interest-scoped — a web Groups tier needs interest-scoping or sample data to populate before launch.

One component

One GroupCard renders both, parameterized on the three axes — mirrors mockup 24's "one blueprint card, one data source" win.

Next step
