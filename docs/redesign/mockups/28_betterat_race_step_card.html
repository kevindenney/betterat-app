Race step card vs standard step card · the timeline view
What a saved race looks like in the Practice timeline, next to an ordinary step. Same card skeleton — a race only adds the on-water grammar it actually carries: course context, a course glyph, and live wind/tide. Built from the real metadata the + composer now persists (race_plan.area_name · course_label · laps).
ModelNo step types — just the is_race flag. A plain step is a neutral status dot + title. A race earns four extra affordances because a race has a venue, a course, marks and conditions an ordinary step doesn't: a race-blue accent + ⛵, an area · course · laps strip, a mini course glyph, and a live wind/tide row that links to the on-water screen. Flip is_race off and all four collapse away.

Standard step — unchanged
Status dot (done / planned), title, optional 📍 location, sub-step progress, capability chips. That's it. ~80% of steps. No course, no conditions, no blue.
Race step — earned additions
1. Race-blue left rail + ⛵ glyph + "Race" pill. 2. Course-context strip: area · course · laps (saved now). 3. Mini course glyph from course_type. 4. Live wind/tide row → "On the water ›" opens Atlas course & marks.
Build order
Now: rail, ⛵, pill, area·course·laps strip — all from existing metadata. Next: course glyph from course_type. Later: live wind/tide + marks via the on-water / weather join (currently dashed placeholder).
