The interest catalog — where curiosity lands

This is the page a new learner arrives at when they pick Nursing from Discover, or when a working nurse follows a link to betterat.app/discover/nursing. It is a category landing, not an institution landing — no shield owns the page. The mobile pass is the primary register; the desktop pass is the same content unfurled. The job is to feel inviting, not academic.

The iOS register grammar set by the Discover · Paths for you canonical holds here exactly: system gray 6 ground, 16 px white cards, SF Pro on the device, the floating four-tab nav, iOS blue for the action, and the coral discovery accent reserved for system recommendation. Hierarchy is deliberate — Get Started cards are the largest object on the page; Capability Frameworks sit at the bottom so curious users find them, casual ones aren’t bombarded.

Two columns, same priorities, more whitespace

The mobile-first scroll is unfolded into a two-column layout: Get Started as a three-up grid (each card preserves the cover, badge, coral-dot signal, and one iOS-blue action), By specialty as a four-up grid of icon + count chips with a dashed more in search tile, Institutions as a five-up grid, Featured blueprints as a three-up grid, and People to follow as a horizontal row of follow cards. Capability Frameworks moves to the right sidebar — same three rows, expanded with source attribution and stats — so the framework is one glance away for power users without competing with the catalog for the main column. The hero Recommended for new nurses eyebrow carries the coral dot once at the top to signal the system’s framing; from there iOS blue owns every action and coral fires only on Get Started signal rows. The page never theme-colors the title — Nursing is a category, not an institution.

The hierarchy is deliberate, and load-bearing

Visitors to betterat.app/discover/nursing are heterogeneous — pre-nursing students, current students mid-program, working nurses scoping CE. Get Started cards are the largest object on the page because the majority who land here are early. Specialty / Institutions / Blueprints / People are roughly equal weight because once a user has self-categorised, those four become the way they navigate.

Capability Frameworks at the bottom is the deliberate hierarchy move. Power users find them; casual users don’t get hit with regulatory acronyms on first contact. On mobile they live below the people strip; on desktop they live in the sidebar — one glance away, never the focus.

Coral semantic, preserved exactly

#D97757 · system recommendation only

The Discover · Paths register pinned the coral discovery accent to system-recommended starting points and nowhere else. This surface inherits that rule literally:

  • Coral dot fires onGet Started card signal rows (mobile + desktop); the title-block eyebrow on desktop (recommended for new nurses).
  • Coral dot does NOT fire onspecialty chips, institution cards, blueprint covers, people cards, framework rows, the active nav tab, the search bar.

iOS blue owns every action button. System gray 5 owns every chip background. The accent does one job here, like in every other register.

Mobile-first — the desktop is the same content unfurled

The phone is the primary register because that is where new nurses land. Sections are stacked with iOS section-eyebrow grammar; strips peek at the edge so the horizontal-scroll gesture is obvious without arrow chrome.

  • HeaderSF Pro Display 34px / 700. Bolder than the Paths register (which is inviting via a lighter weight) because this surface is declarative — it names a category.
  • Section titlesSF Pro Display 22px / 700 as a small head, with a 13px gray secondary lede line.
  • Get Started cardfull-bleed 144px cover, badge top-left, mono cover-tag top-right, serif topic title at the bottom of the cover. The cover image is a placeholder until shoot.
  • Specialty chip36px pill, leading glyph in gray-3, trailing count in gray-3. Wraps freely.
  • Strip cards156–220px wide, peek at right edge, no arrow.
  • Capability rowgrouped iOS list, 44px tag icon with the framework name set in serif, hairlines between, chevron right.

What this surface deliberately doesn’t do

  • No hero photo on the titleNursing is a category, not an institution. A full-bleed hero photo would push it toward editorial-magazine; the surface should feel like a catalog one can scan.
  • No category color on the titlethemed title bars happen on org pages (where one school owns the surface). Categories are interest groupings — they stay neutral so they don’t pre-bias the user toward any one specialty.
  • No tabs across the topthe section list IS the navigation. Tab chrome for Paths · People · Orgs would compete with the cards for the eye on first contact.
  • No progress percentages anywherethis is an interest page; a user has not started anything yet. Counts (12,400 nurses started) are population signals, not personal-progress signals.
  • Search opens at the topthe search glyph in the back row (mobile) and the ⌘K input (desktop) are gateways — the actual search surface is its own canonical, not nested here.
  • Capability Frameworks isn’t a CTAthree quiet rows; one chevron each. The action verb on this page is begin a sequence, not read the standard.
