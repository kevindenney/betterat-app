Design notes

Why these choices

Four moves the canonical commits to. Each one is small enough that Claude Code can implement it without further design back-and-forth — but each is load-bearing for the May 20 launch.

One URL, three states

The page is not three different pages. Same hero structure, same author attribution, same Plan/Do/Reflect coda. State B is State A with a personal headline and a verb-explicit primary action. State C is invisible — a server-side 302 — so most existing-followers never see the page at all.

The HKDW host chrome shown explicitly

The dark navy bar at the top of the mobile frames is the HKDW app's webview chrome, not ours. Rendering it in the canonical makes it explicit that sailors arrive from inside HKDW — which constrains our hero (no top-of-page logo to compete, no “jump to homepage” affordance, the close X is HKDW's not BetterAt's).

No stock-imagery

The brief asked specifically: no generic team-of-professionals photography. The hero uses a single Dragon-class sail abstraction — two triangles, navy mast — as watermark on mobile and a stage element on desktop. A real line illustration or RHKYC archive photo can drop in later without restructuring the hero.

Kevin's authorship is the second thing you read

The eyebrow names the event. The headline names the goal. The author strip — blueprint by Kevin Denney — is the third beat. Sailors who don't know BetterAt should know whose practice they're opting into before they tap the primary CTA.
