1 · One step-card grammar, three by-lines.

Subscribed blueprint, co-practitioners, followed person — all three render the same ts-card pattern (state pill · title · meta · capability chips · add button). What varies is the by-line at the top (author avatar + name + role + follow state). This is how a user reads whose timeline am I looking at without needing chrome to tell them.

blueprint peers follow

2 · On deck buys the user permission to capture without committing.

The biggest psychological cost of adding from someone else's timeline is "where does it fit?" — and that question stops capture. The deck zone removes the question: "I'll figure out where it fits later, just hold it." The amber strip lives above the timeline so the user can see what they've collected without it cluttering their committed plan.

amber strip · provenance per row · Place → / ×

3 · Sharing a step is forking, not editing.

When someone shares a step with you, they're saying "look at what I'm doing — copy it if it's useful." Receivers can view, fork into their own timeline (with provenance pointing back), or comment. They cannot edit the original. This keeps authorship clean and means receivers experience shared steps the same way they experience blueprint steps — as a source to fork from.

view · fork · comment — never edit

4 · Fleet view is read-only by design.

14 sailors raced; 14 sailors wrote what they saw. The fleet view threads them together by timestamp so you can read what happened from multiple angles, but you can't edit anyone else's words or capabilities. Every capture still belongs to its author. The value is in the parallel testimony, not in collaborative editing.

your captures others' · timestamp-threaded · read-only

5 · Timing was never the model — only the chrome.

Captures have always been wall-clock timestamped; that doesn't change. What retires from the surface is the elapsed counter as a default chrome element. Sailing keeps it on (races are inherently timed). Nursing, drawing, reading — none of them benefit from a timer ticking up while the user is trying to capture an observation. The toggle stays one tap away for the user who wants it, but it doesn't take screen real estate from people who don't.

state pill stays · timer becomes optional

6 · Interest-flavored language without per-interest components.

Sailing's Live · capturing · Nursing's On shift · capturing · Drawing's Session · capturing · Generic Practicing — same coral pill, same anatomy, different string. Same for the Stop CTA: Stop capturing / End shift / End session / Done capturing. Configuring this per-interest is two strings in a JSON file — and it makes BetterAt feel native to whatever practice a user shows up to. Nursing users don't read "Live · capturing" and think the system is built for sailors; they read "On shift" and feel known.

one component · interest-flavored labels
