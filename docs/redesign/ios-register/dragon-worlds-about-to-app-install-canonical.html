From the Dragon Worlds event app About page to BetterAt installed on your phone — a five-phase journey

The screenshot the team is iterating on is the BetterAt-authored About surface that lives inside the host event app for the Hong Kong Dragon Worlds 2027. From here, a verified competitor needs to (1) tap into the redeem, (2) accept the offer on web, (3) land on their first Practice step in mobile Safari, (4) install the native BetterAt app, and (5) re-open in native with the blueprint already subscribed. We treat the journey as web-first: the user never has to install before getting value, and the install happens only after they've felt the win. The whole flow can complete in under two minutes; the first three phases work without any download at all.

ABlueprint index

How they see and add the rest of the blueprint

From the step strip at the top of Plan view, tap the blueprint title to expand. The full 12-step list appears, grouped by status: done (green checks, locked), current (blue ring, no Add button), upcoming (gray, each with its own + Add button).

Adding an upcoming step queues it into the user's Plan timeline at its native blueprint position; it doesn't jump the current step. The list is a menu of the future, not a checklist to grind.

step.add({ blueprintStepId, position: native })

BFleet plans

How they see what other subscribers are doing

From the WITH-row chip ("Worlds Fleet · 63 sailors"), tap to enter the fleet view. Each peer card surfaces their current step, their last activity, and a progress mini-bar. Status chips name the relationship: Same step (blue), Ahead (green), Behind (gray).

Tapping a peer opens their public timeline for this blueprint — their notes, evidence, and reflections, scoped to the same blueprint. Private notes stay private; published reflections are shared.

fleet.peers({ blueprintId, sort: 'currentStep desc' })

CStep discussion

How they see & comment on the same step

Every step gets a Discussion tab alongside Plan / Do / Reflect. The feed gathers notes from all sailors on this step in the same blueprint, with evidence chips (voice clips, photos, polar data) inline. Reactions are three named verbs — fire, insight, question — not generic likes.

The blueprint creator (Kevin) gets a coach pip and his replies are visually pinned beneath the note he replied to. The discussion is scoped to the step — you don't have to wade through the whole blueprint feed to find help on what you're working on right now.

step.discussion({ stepId, includeCoachReplies: true })

What happens between phases · technical & UX notes

The system carries one durable thing across all five phases: a single-use redeem token, scoped to the event-app user id, valid until November 22, 2027. Everything else — the welcome copy, the smart-app banner, the universal link — hangs off that one identifier.

1About → tap

Mint a token, open the URL

The event-app deep-link handler mints a single-use token tied to the verified competitor's id and event registration. The CTA opens https://better.at/r/[token] in the system browser. If BetterAt is already installed, iOS Universal Links intercept and jump straight to phase 5 — phases 2–4 are skipped silently.

betterat://r/[token] · UL · fallback Safari

2Safari · redeem

One CTA, no form

The redeem URL renders the welcome surface. Accept & start preparing creates a session-level account (no email, no password) tied to the token, subscribes the user to the blueprint, and 302s to the first-action URL. The token is burned on success.

POST /api/redeem · session 90d · email-claim optional

3Plan view · web

Step 1 pre-populated

The same Plan-tab component the native app ships renders here as web. Welcome toast names Kevin and the fleet. The smart-app banner is pinned at the very top of the viewport on every better.at page the user visits, dismissible × persists for 7 days. The user can complete the whole journey here without ever installing.

SmartAppBanner · localStorage · 7-day re-show

4App Store

iOS owns the chrome

Tapping Install on the smart-app banner (or tapping Do tab in web) raises the standard iOS App Store sheet. Our copy lives in the App Store listing — the three "why" lines name the native-only capabilities (mic · camera · offline). After install, the standard iOS resume flow opens the app at the same Universal Link.

apps.apple.com/app/betterat · UL · /practice/step/[id]

Three principles · why this flow stays short

1 · Value before account

The user reaches their first useful action — Step 1 of Kevin's blueprint, pre-loaded in Plan view — without ever filling a form. No email, no password, no card. The session-level account is created silently on Accept; email-claim is offered later, only if and when the user wants to log in from another device.

session-level account

2 · Install is opt-in, not the gate

Phases 2 and 3 are entirely web. The smart-app banner is the only install ask, and it surfaces only after Step 1 is already loaded. The user installs because they want voice capture on the water, not because we forced them to before they understood what BetterAt was. Friction is paid at the right moment.

smart-banner · Do-tab sheet

3 · State follows the user

The redeem token is the durable handle. Whichever device the user lands on next — same phone after install, a laptop at home, a new phone next month — opening better.at/r/[token] (or its Universal Link) resolves to the same session, same blueprint, same step. Re-onboarding never happens.

universal link · session sync
