Close, then hand off

The temptation in a post-reflection screen is to dwell — confetti, celebration cards, "Great job!" The whole loop suffers from that. If Reflect's reward state lingers, the next Plan never starts.

So we close in three lines (banner, stats, digest) and immediately give the biggest visual weight to carrying forward. The user's eye lands on the green banner, slides through the stats, reads the two-line digest, and ends on the carry-forward card. That's the gradient: done → done → done → next.

green · close   blue · forward

Build on / Watch for

The two bullets inside the carry-forward card are deliberately asymmetric in color: Build on is green-deep (extend what's working); Watch for is iOS-coral (the only loud-warning use of coral on this surface). Coral signals "this is the thing that bit you last time" without screaming.

The 56px tag gutter on the left makes the two bullets read as a coaching brief — name the thing, then the instruction. Same shape as a sailing race-debrief whiteboard.

coral · watch   green · build

One celebrated metric

Three stat tiles, only one is washed green — the Capabilities advanced tile. That's not arbitrary. The whole point of the BetterAt loop is to grow capability over time; "5 of 6 prompts answered" is process, "3 captures cited" is hygiene, but "2 capabilities advanced" is the signal.

Celebrating only the signal teaches the user, screen by screen, which number to care about — without ever printing the word "matters."

green wash · only on the metric that matters
