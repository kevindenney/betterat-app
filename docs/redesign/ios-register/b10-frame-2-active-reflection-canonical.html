Three row states on one surface

This frame is the only place in the app where the user sees three distinct prompt-row states at once: collapsed, answered, and active. They share the same anatomy (left stripe, prompt text, optional chev/field/answer/chip stack) and differ only in chrome — stripe weight, border presence, shadow softness, focus halo. The user learns the whole row vocabulary by scanning a single screen.

green · answered   blue · active   gray · collapsed

Pattern surfacing lives between sections

The pattern card is not a prompt — it's an AI observation. Placing it between What worked and What to improve keeps it from competing with the user's own answers, and the small tab-flag at the top reads as a different object class. It only appears when the AI has at least two captures that hang together; one capture, no pattern.

It's also dismissible by ignoring — there's no Accept/Reject CTA. Patterns are observations, not decisions.

green · pattern card flag

The hint as a contract

The disabled CTA at the bottom never simply greys out — it always pairs with a hint that names exactly what's still required. "Need one What to improve answered to activate." That's a contract: do this one thing, and the CTA lights up.

This pattern keeps Reflect from feeling like a quiz with a hidden rubric. The user always knows the next move.
