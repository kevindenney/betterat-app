The quiet moment of "I notice I did the thing" — a capture promotes itself into evidence of capability

From the post-activity stream the user has tapped a capture (the 14:23 voice clip about the eight-degree shift) and chosen Mark as evidence. A bottom sheet slides up over a dimmed Frame 3, blurred just enough to keep context legible. The capture rides at the top — same row vocabulary as the stream, but framed in a soft green wash that says "this one is being promoted." Below it: a multi-select list of capabilities from the active blueprint, each row a tappable name + brief description + tiny progress pip. A three-step strength selector gives the user a way to say "this was a real one" without forcing the choice. The Save CTA stays iOS-blue — saving evidence is still a forward action, not a decoration.

Tone: slightly celebratory but not loud. The green is restrained, the language is plain ("Evidence for which capabilities"), the affordances feel like a checklist rather than a trophy.

Anatomy · 393 × 852 + spec column

Measurement chips in amber over the bottom-sheet shell, the multi-select row, the segmented control, and the CTA stack. Callouts A–G align with Frames 2 & 3 — A is the sheet shell (frame's "state container"), G is the primary CTA + dismiss pair.

Green is the only reward

The emotional payload of this frame is a single color shift — gray-bordered checkbox to green-filled checkbox, white row to green-wash row. No confetti, no spring physics on the count, no copy that congratulates. The user is doing a piece of light bookkeeping that happens to also be meaningful; the design respects that by not over-dressing it. The same restrained green carries over from Frame 3's Activity complete pill — it's the same vocabulary saying the same thing: "noticed."

The capture stays the capture

The row at the top of the sheet is the same row the user just tapped on in the stream — same radius, same left-accent, same waveform, same chip, same italic body for voice. Reusing it verbatim signals this is the moment we're talking about with zero new vocabulary to learn. The green wash and the "Promoting this capture" eyebrow are the only signals it's been lifted out of context.

row contract preserved · section frame

Multi-select is the truth of the model

A single observation usually evidences more than one capability — the eight-degree shift call is both reading wind shifts and pre-start positioning. Forcing a single choice would compress the signal. The pip + stage on each row makes the consequence visible: tapping a capability isn't just a tag, it's pushing a named line one notch up its progression bar. The strength selector is optional on purpose — the model can survive ungraded evidence, and asking too much is what kills the habit.
