Long-press menu

The pressed row lifts above a blurred scrim; an iOS-native context menu pops below it. Eight actions, grouped: edit / scheduling, then sharing, then a single destructive delete in system red.

One gesture, two scales of meaning

The zoomed-in carousel renders the current step in full detail; the zoomed-out list renders the full practice history as one continuous timeline. The pinch is the only way to switch — no tab, no toggle, no segmented control. Both views share the same header strip and tab bar, so the user always knows which interest they're in and where the FAB lives. The chrome is continuous; only the body content rescales.

iOS blue accent · 56pt FAB persists across zooms

Sections are sticky and time-relative, not date-bucketed

The user thinks in tomorrow / today / this week / last week / earlier in May, not in calendar months. Section headers stick to the top as the user scrolls, so the temporal context is always visible. The NOW divider is a thin blue rule with the live date — it is the only horizontal line in the entire list, and it marks the user's place even when scrolled away. Each row shows one relative-time accent (in 3 days · tomorrow · today) so the meaning is legible without parsing the date.

NOW · sticky headers · relative-time accents

Three dots per row tell the whole story of that step

Every row carries the three-phase status — Plan / Do / Reflect — as three dots. filled means done, half means in-progress, empty means not yet. Completed steps go green-filled. Long-press on any row pops the iOS context menu; delete is the only destructive option, always last, always system red.
