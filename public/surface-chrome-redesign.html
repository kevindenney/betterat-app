Drill-down chrome — kill the double header

The same stacked “interest pill + Back” we just fixed on Fleet shows on other deep-detail routes.

Unlike the Fleet hub, these are deep details (a specific race, a course, a boat). Switching your active interest from inside one of them is odd — so the fix here is simpler than Fleet: drop the redundant global interest header entirely and keep only the screen’s own Back. No inline switcher.

Surfaces this applies to

  • Race detail & race flows — /race/[id], /race/add-tufte, /race/scrollable/[id], /race-session/[sessionId]
  • Learning — /my-learning
  • Boats — /boats, /boat/add, /boat/edit/[id]
  • Tuning guides — /tuning-guides (also double-pads the top: it adds insets.top + 8 on top of the global header)

How it’s fixed

  • One-line change per family. Add the route prefix to ROUTES_WITH_CUSTOM_TOOLBAR in app/(tabs)/_layout.tsx (e.g. /race, /boat, /boats, /my-learning, /tuning-guides) so the global interest header stops rendering on native — exactly what we just did for /fleet.
  • Each screen keeps its own Back. They already render one; just confirm each owns its top safe-area inset (insets.top) once the global header is gone. /tuning-guides already does — it’ll simply stop being double-padded.
  • No inline interest switcher here. That’s reserved for hub screens like Fleet. Deep details inherit the interest you’re already in.
