Decisions this mock encodes

1. Craft-first. Screen 1 replaces the sailing showcase's front door; the pick is cached via the existing onboarding_interest_slug AsyncStorage key (already read post-signup), so the funnel exit lands the new account in the right interest with zero extra steps. "I'll decide later" keeps the guest path alive.
2. One story, data-driven copy. Screens 2–3 are ONE component pair fed by an onboardingVocab[slug] table (loop line, plan/do/review examples, peer/club nouns) — the same pattern as interestVocab. Adding a craft = adding a row, not a screen. Unknown/"something else" picks fall back to universal copy.
3. The story beats mirror the product's real idioms — sequence-to-anchor with the NOW divider (screen 2), watch → adapt-to-my-practice + learner-pull subscribe (screen 3). No invented features, no fake social proof (launch truth-state).
4. Killed: track-races.tsx, prepare-pro.tsx, join-crew.tsx and their sailing animations — replaced by pick-craft.tsx + two vocab-driven screens. Route: welcome-back → value/pick-craft → value/loop → value/people → auth-choice (unchanged tail).
5. Illustration style: static product-shaped vignettes (cards above) instead of bespoke animated scenes per craft — the loop card IS the product, so the funnel previews reality instead of a cartoon.
