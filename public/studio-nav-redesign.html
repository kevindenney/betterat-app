One shell — Practice ⇄ Studio

The real problem isn't the dropdown — it's that practitioner and creator are two separate apps today, bridged by a cluttered profile menu. Every fix below follows one rule: one workspace, two contexts, no duplicate doors.

① The model ② Option A — shared left rail ③ Option B — Do · Create toggle ④ Cleaned profile menu ⑤ Public face = profile (one screen) Summary
The model
Two contexts inside one frame, not two apps

You're always in one workspace (Personal, or an org like JHU where you're admin). Inside it you flip between Practice (do & learn) and Studio (create & mentor). "Mentor" was never a third thing — subscriber threads & cohorts are just Studio's audience side, so they fold in. Switching context becomes a single click in the same frame — never a jump to a different-looking app.

Practice context — your tabs today (Practice · Library · Watch · Atlas) Studio context — blueprints · subscribers · threads · payouts Inbox — promoted out of the menu into the chrome
Option A · recommended for web
Shared left rail — Practice and Studio live in the same shell

The rail never changes shape. Workspace switcher up top, two labelled groups below, you & Inbox at the bottom. Going Studio→Practice is just clicking a nav item — the "Profile → Creator Studio" round-trip disappears.

What I'd actually ship

  • 1
    Pick Option A for web, Option B for phone. Same principle — one frozen frame, contexts swap inside it — expressed as a left rail on wide screens and a Do·Create toggle on narrow.
  • 2
    Studio is the single home for create + mentor. Delete "Cohorts I mentor" and "Subscriber threads" from the menu; they're already Studio's Subscribers & Threads.
  • 3
    Promote Inbox to the chrome (rail footer / header bell) with its badge. It's a capture surface, not an account setting.
  • 4
    Drop "My Practice" from the menu. Keep the /practice deep-link for marketing CTAs; the menu entry is a duplicate once Practice is a permanent context.
  • 5
    Merge public face + edit into one Profile with a Preview ⇄ Edit toggle. Preview is literally the stranger's render; Edit makes the same sections live.
  • 6
    Make the workspace switcher real. Personal vs JHU should swap the shell's data/permissions — that's the one menu item worth keeping prominent.
