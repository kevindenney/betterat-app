Resource detail — trimming the oversized cover

The current screen opens with a 16:9 "book cover" box. For a PDF with no thumbnail that's a large empty rectangle showing only a colored spine + "PDF" stamp — it pushes the actually-useful sections (Read/Listen, where it appears in your practice, your marks) below the fold. Four options below, from lightest-touch to most ambitious. Mock only — nothing wired.

Option A — Compact file band

Lightest touch

Swap the 16:9 box for a slim file row: small spine-thumb, title/meta, and a primary Open. Recovers ~60% of vertical space; useful sections rise above the fold.

Option B — Real inline preview (web)

Turns dead space into content

On web, render the actual first page in an iframe (same trick as the YouTube embed) at a reduced height, with Open full. Native keeps the compact band from A. Dead box becomes a real preview.

Option C — Collapsed strip (tap to expand)

Keeps a cover, optional

Shrink the cover to a 60px strip by default; tap Preview to expand to the full cover/preview. Best if you sometimes want the visual but not always.
